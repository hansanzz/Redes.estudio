Direccionamiento IP

La dirección IP es una dirección lógica (no física, como la MAC): se le asigna a cada equipo, no viene "de fábrica" grabada en el hardware, y puede cambiar según a qué red te conectes. Nunca puede haber dos equipos con la misma dirección IP dentro de una misma red al mismo tiempo, porque generaría un conflicto.

¿Cómo está formada una dirección IP?

Una dirección IP (versión 4) está formada por 4 bytes, es decir, 32 bits en total. Cada byte se escribe en formato decimal, separado por puntos: por ejemplo, 192.168.1.10.

Con 8 bits, el número más grande que se puede formar es 255 (en binario: 11111111), y en total se pueden formar 256 números diferentes (del 0 al 255), porque 2 elevado a la 8 es igual a 256.

Cada dirección IP se puede dividir conceptualmente en dos partes:

  • Parte de red: identifica a qué red pertenece el equipo.
  • Parte de host: identifica a ese equipo específico dentro de esa red.

Clases de red: A, B, C, D y E

Existen 5 clases de direcciones IP. Las primeras tres (A, B y C) se usan para asignar direcciones a redes y equipos; la D y la E tienen usos especiales.

Clase A

Pocas redes, pero cada una con muchísimos hosts posibles. La usan organizaciones enormes, como gobiernos, que tienen pocas LAN pero cada una con muchísimos equipos conectados.

Clase B

Un equilibrio entre cantidad de redes y cantidad de hosts por red. La suelen usar empresas medianas, que necesitan varias subredes con una cantidad moderada de equipos en cada una.

Clase C

Muchísimas redes posibles, pero cada una con pocos hosts. La usan usuarios particulares y redes domésticas: representan muchas redes chicas, cada una con pocos equipos conectados.

Clases D y E

La clase D se usa para multicast (enviar un mismo mensaje a un grupo específico de receptores). La clase E está reservada para uso experimental y de investigación, no se usa en redes comerciales.

Broadcast y Multicast

Broadcast es cuando un mensaje se envía a todos los equipos de una red al mismo tiempo. Una dirección IP de broadcast se identifica porque tiene todos los bits de la parte de host en 1 (es decir, en 255 en esa parte de la dirección).

Multicast, en cambio, envía el mensaje solo a un grupo específico de receptores que se suscribieron a esa comunicación, no a todos los equipos de la red. Usa las direcciones de clase D.

Por otro lado, una dirección IP que tiene todos los bits de host en 0 no identifica a un equipo en particular, sino que identifica a la red entera (es la "dirección de red").

Para seguir aprendiendo: el router

El dispositivo de capa 3 que conecta distintas redes y maneja direcciones IP es el router. Puede separar dominios y dividir una red grande en redes más pequeñas, llamadas subredes locales.

Cuando los nodos de una red están conectados por múltiples trayectorias posibles, es el router el que elige el mejor camino para cada comunicación, usando protocolos de enrutamiento para decidir la ruta más eficiente en cada momento.
