Modelo TCP/IP

Mientras que OSI es un modelo teórico de 7 capas, TCP/IP es el modelo que realmente se usa en Internet: tiene solo 4 capas y agrupa varias funciones de OSI en una sola. Es "menos prolijo" en la teoría, pero es el que de verdad mueve cada dato que viaja por la red.

Las 4 capas del modelo TCP/IP

Cada capa de TCP/IP agrupa una o más capas de OSI. Compará ambos modelos en esta tabla:

Modelo TCP/IP (4 capas)Equivalente en OSI (7 capas)
AplicaciónAplicación + Presentación + Sesión
TransporteTransporte
InternetRed
Acceso a la red (host a red)Enlace de datos + Física

Es donde funcionan los protocolos que usan directamente los programas: navegar la web, mandar correos, transferir archivos. Algunos protocolos y su función:

  • HTTP: transferencia de páginas web. Usa el puerto 80 (y 443 para HTTPS).
  • FTP: transferencia de archivos entre computadoras.
  • SMTP: envío de correo electrónico.
  • POP3 / IMAP: recepción de correo. POP3 descarga los mensajes; IMAP los mantiene guardados en el servidor de correo, para poder verlos desde varios dispositivos.
  • SNMP (Simple Network Management Protocol): permite administrar y monitorear dispositivos de red de forma remota.

HTTP vs HTTPS: HTTPS es HTTP con una capa extra de encriptación (usando el protocolo SSL/TLS), que protege los datos que viajan entre el navegador y el servidor para que nadie pueda espiarlos ni modificarlos.

TELNET vs SSH: ambos permiten conectarse de forma remota a otra computadora usando una terminal, pero SSH es mucho más seguro porque encripta toda la comunicación, mientras que TELNET la envía sin proteger.

Proporciona servicio de comunicación confiable de extremo a extremo (con TCP) o servicios de comunicación sin garantía de entrega (con UDP). Esta capa segmenta y ordena los datos, multiplexa las conexiones simultáneas y define cómo dos aplicaciones mantienen una conversación sobre el protocolo IP.

Dos funciones clave de esta capa: dividir los datos en segmentos manejables, y controlar que la información llegue en el orden correcto (en el caso de TCP).

Se encarga del direccionamiento lógico, del enrutamiento, de la fragmentación de paquetes y de su reenvío. Pensalo como el "cartero" de la red: así como el correo recibe una carta y una dirección de destino, esta capa recibe de la capa de transporte un paquete de datos (equivalente a la carta) junto con una dirección IP de destino (equivalente a la dirección postal).

Protocolos de esta capa:

  • IP (Internet Protocol): asigna y gestiona las direcciones lógicas.
  • ICMP (Protocolo de Control de Mensajes de Internet): si hay un problema al enviar un mensaje, ICMP le avisa al emisor con un mensaje de error (es lo que usa el comando ping).
  • ARP (Address Resolution Protocol): obtiene la dirección MAC (física) a partir de una dirección IP.
  • RARP (Reverse ARP): hace el proceso inverso, obtiene una dirección IP a partir de una dirección MAC.

Maneja cómo los datos se transmiten físicamente. Usa un protocolo distinto según el medio: por ejemplo Ethernet para conexiones por cable, y protocolos ópticos específicos para conexiones por fibra óptica.

TCP vs UDP: ¿cuál usar según la situación?

SituaciónProtocolo recomendadoPor qué
Enviar correo electrónicoTCPImporta que todos los datos lleguen completos y en orden, la velocidad no es lo prioritario.
Video o llamadas en vivo / juegos onlineUDPImporta más la velocidad y la baja latencia que la entrega perfecta de cada paquete.
Menor latenciaUDPNo pierde tiempo confirmando la recepción de cada paquete.

Protocolos y sus puertos

Los puertos son números que identifican a qué aplicación pertenece cada dato que llega a un equipo. Así, una misma computadora puede tener el navegador, el correo y otros programas usando la red al mismo tiempo, sin que los datos se mezclen.

ProtocoloPuertoFunción
HTTP80Navegación web
HTTPS443Navegación web segura
FTP21Transferencia de archivos
SMTP25Envío de correo
POP3110Recepción de correo
